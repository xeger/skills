# Individual Goal Setting

## Overview

Most draft goals describe **activity** ("ship the dashboard") rather than **output** ("60% of target customers using the dashboard weekly"). This skill refines a goal by pressure-testing it against a small set of principles and criteria, then rewrites it as a measurable outcome.

Applies to rocks (EOS), OKRs, KRs, quarterly objectives, annual goals — any individual or team commitment that needs to be both motivating and measurable.

## Requirements

Before producing an enhanced goal, you MUST:

1. Map the user's input to the 11 worksheet questions and identify which are answered, partial, or missing.
2. Prompt the user for the missing answers — never invent them. A goal built on assumed numbers, baselines, or commitments is worse than the original draft.
3. Verify the rewritten goal satisfies every row of the criteria checklist. If any row fails, the goal is not ready — say so and continue refining.
4. Show the filled-in worksheet alongside the enhanced wording. The reasoning is part of the deliverable, not optional context.

## Principles

- **Less is more.** Fewer goals, sharper goals.
- **Output, not activity.** Measure what changes for users/the business, not what the team does.
- **Leading, not lagging.** Embed indicators visible early enough to course-correct.

## Criteria checklist

A refined goal MUST satisfy all five. If any row fails, the goal is not ready.

| Criterion | Test |
|-----------|------|
| **SMART** | Specific, Measurable, Achievable, Relevant, Time-bound |
| **Relevant** | The output is within the team's direct influence |
| **Leading indicators** | Early signals exist and are named |
| **Commitment clarity** | Marked as must-do (100%) or aspirational (~70%) |
| **Mission/company alignment** | Links to team mission and a company-level goal |

## Workflow

1. **Take the input.** User pastes a draft goal and may pre-answer some of the questions below.
2. **Identify gaps.** Map their input to the questions. Note which are answered, partially answered, or missing.
3. **Prompt for the missing ones.** Ask only what's missing — do not re-ask answered questions. Batch related questions in one prompt to keep it fast.
4. **Generate the enhanced goal.** Produce a single sentence (or two) that's SMART, output-focused, time-bound, with a leading indicator and a commitment level.
5. **Show your work.** Return the filled-in question set alongside the enhanced wording so the user can see the reasoning.

If the input is rich enough that all questions are essentially answered, skip step 3 and go straight to the enhanced wording.

## The questions

Use these as the refinement worksheet. Skip any the user has already answered.

1. **Current wording** — what's the draft goal as written?
2. **Intent & mission link** — why does this goal exist? How does it connect to personal or team mission?
3. **Company-goal alignment** — which company-level goal does this ladder up to? *(For personal or broad-scope goals where no company goal applies, say so explicitly — never silently skip.)*
4. **Tightening** — where is the wording vague, hedged, or open to interpretation?
5. **Activity → output** — what's the *outcome* this activity is supposed to produce? Restate the goal in those terms.
6. **User/customer impact** — if achieved, what concretely is better for the people the team serves?
7. **Business impact** — what does the business get? (retention, revenue, cost, risk reduction, capability)
8. **SMART tightening** — what specific numbers, dates, and owners are missing?
9. **Leading indicators** — what early signals (within the first 1–4 weeks) will tell you this is on track?
10. **Commitment level** — must-do (100%) or aspirational (~70%)?
11. **Enhanced wording** — produce the rewritten goal.

## Examples

### Example 1: Activity → measurable adoption

- **Draft:** Ship new reporting dashboard
- **Intent:** Customers struggle to access insights and decide quickly; team mission is to empower users with clear, actionable data.
- **Activity → output:** Move from "ship dashboard" to adoption, usage frequency, and behavior change.
- **User impact:** Customers make faster, more informed decisions; less reliance on support.
- **Business impact:** Higher retention, increased feature adoption, upsell to higher tiers.
- **SMART tightening:** Add adoption %, usage cadence, deadline.
- **Leading indicators:** First-2-week active users, dashboards created per user, repeat visits.
- **Commitment:** Must-do — core to product value.
- **Enhanced:** *Launch the reporting dashboard and reach 60% adoption among target customers within 60 days, with at least 40% of adopters using it weekly by end of Q3.*

### Example 2: Vague improvement → activation metric

- **Draft:** Improve onboarding for new customers
- **Intent:** New customers take too long to see value, hurting retention and expansion; team mission is to drive customer value quickly.
- **Activity → output:** Move from "improve onboarding" to activation rate and time-to-value.
- **User impact:** Faster adoption, fewer stalled accounts.
- **Business impact:** Higher retention → ARR growth and better LTV.
- **SMART tightening:** Activation %, time-to-value in days, deadline, owner.
- **Leading indicators:** 7-day product activation, onboarding completion rate, first key action taken.
- **Commitment:** Must-do — directly tied to revenue and retention.
- **Enhanced:** *Increase new-customer activation rate from 55% to 75% within 30 days of onboarding by end of Q3, reducing average time-to-value from 21 days to 10 days.*

## Common Mistakes

| Mistake | Result | Fix |
|---------|--------|-----|
| Goal names a deliverable ("ship X") | Team can ship the thing and still fail to deliver value; nobody can tell whether the goal "worked" | Always restate as the change the deliverable causes |
| Metric is lagging only (ARR, churn) | Quarter ends before you know you're off-track; no time to course-correct | Always add a leading indicator visible within weeks |
| No deadline or fuzzy deadline ("by end of year-ish") | Goal silently slips; no forcing function for prioritization | Pin a date; treat slippage as a separate decision, not a quiet drift |
| Numbers without a baseline ("reach 75% activation") | Cannot tell if 75% is a stretch or a regression; no way to size the lift | Always state the current level: "from 55% to 75%" |
| Aspirational goal mislabeled as must-do | Either the team burns out chasing 100% on a stretch, or the org loses trust when "must-dos" miss | Mark commitment level explicitly; ~70% is fine when labeled |
| Goal depends on another team's work | Outcome decided by a team you don't control; you can do everything right and still miss | Narrow scope to your team's direct influence, or name the dependency as a stated risk |
| Inventing baselines or targets to fill in the worksheet | Goal looks crisp but rests on fiction; review converges on fake numbers | Never fabricate — ask the user for the real figures, or mark "TBD: needs baseline" |

## When NOT to use

- Status updates or progress reports — this is for goal authoring, not tracking.
- Hard deadlines imposed externally (regulatory, contractual) where the goal is already specified — refinement won't change the constraint.
- Tasks under ~2 weeks — this framework is overkill; just write the task.
